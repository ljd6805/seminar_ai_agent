# 발표자 노트 — AI Agent 입문 세미나 (54장 · 90분) v1.1

각 항목 구조: **한 문장**(이 장의 핵심) / **풀어서**(말로 할 설명) / **우리 일로**(설계·검증 연결) / **주의**(오해·한계).
슬라이드 번호는 v1.8(54장 = 본편 40 + 부록 14) 기준. 부록(41~54장)은 발표 시 넘기지 않는다.

## 발표 전체를 관통하는 것 (고정)

> **AI Agent는 사람의 판단을 대체하는 도구가 아니라, 판단 전에 필요한 반복 작업을 줄이는 도구다.**

- **관통 시나리오**: Regression Triage Agent 하나를 — 데모 1(prompt만) → 데모 2(Skill) → 데모 3(MCP) → 데모 4(Harness) → 사람 승인으로 키운다. "다음 단계로 넘어갈 때마다 무엇이 부족했는지"가 서사의 엔진.
- **4요소(3장, 동등 4행)**: Agent(생각·실행) · Skill(방법·규칙) · MCP(연결) · Harness(실수해도 안전하게) — 질문으로 기억: 어떻게 움직이나 · 무엇으로 일하나 · 무엇과 연결되나 · 어떻게 통제하나.
- **4원칙(12장)**: 작게 시작 · 출력 구조화 · 근거 남기기 · 사람 승인 — 이후 배지가 보일 때마다 한 번씩 짚는다.

## 반드시 바로잡을 오해 6 (해당 장에서 명시적으로 언급)

1. LLM은 항상 사실을 말하는 것이 아니다 → 8·10장
2. Agent는 무제한으로 알아서 도는 것이 아니다 — 권한·한도·확인 안에서 → 6장
3. 사내 AI라도 접근 권한·프로젝트 경계는 지켜야 한다 → 14·16장
4. MCP는 연결 규격이지 보안 정책 자체가 아니다 → 25·26장
5. schema validation은 내용의 진실성을 보장하지 않는다 → 32장
6. Harness는 모델을 믿기 위한 장치가 아니라 모델 사용을 통제하기 위한 장치다 → 31장

추가(v1.1): "사내망에서 접속했다 ≠ 데이터가 사내에서 처리된다" → 14장에서 반드시 입으로.

## 3장 · 오늘의 결론 · 4요소

- **한 문장**: 결론을 3분 안에 — 반복은 Agent, 판단은 사람. 그리고 오늘의 네 단어.
- **풀어서**: 네 줄을 우측 질문으로 읽는다 — "Agent는 어떻게 움직이는가, Skill·MCP로 무엇을 하고 어디에 닿는가, Harness로 어떻게 통제하는가. 오늘 이 질문들에 답하고 끝납니다."
- **주의**: 용어 설명을 시작하지 말 것 — "낯설어도 됩니다"로 바로 데모.

## 4장 · 데모 1 (prompt만)

- **한 문장**: regression 로그 분류, 대화만으로 여기까지 — 단 원인 불명 3건이 남는다.
- **풀어서**: 성공 데모가 아니라 "되는데 완벽하지 않은" 데모임을 명시. unknown 3건을 일부러 짚는다.
- **주의**: 구성된 예시 장면 라벨 언급. 로그 3파일·12건 등 디테일은 illustrative.

## 5장 · 데모 1 해부

- **한 문장**: 계획·도구·확인이 다 들어 있었다 — 위 과정을 하나로 묶으면 AI Agent의 정의가 된다.
- **풀어서**: 슬라이드엔 없지만 말로 훅 질문을 던져도 좋다 — "원인 불명 3건이 남았죠. 이 결과를 실무에 넣으려면 무엇이 더 필요할까요? 오늘 본문이 그 답입니다."

## 6장 · AI Agent 정의

- **한 문장**: Agent는 정해진 권한과 한도 안에서 이해→계획→실행→확인을 반복하는 실행자다.
- **주의**: "될 때까지 알아서 돈다"처럼 들리지 않게 — 권한·한도·확인을 말에 넣는다. (오해 ②)

## 7장 · 챗봇 vs Agent

- **한 문장**: 질문에 답하면 챗봇, 목표를 받아 도구를 쓰며 진행하면 Agent에 가깝다.
- **주의**: Agent가 항상 우월하다고 말하지 않는다 — 단순 질의응답엔 챗봇으로 충분.

## 8장 · LLM

- **한 문장**: LLM의 기본 동작은 다음 말(엄밀히는 다음 토큰) 예측 — Agent는 여기에 도구·반복·검증을 결합한다.
- **풀어서**: 하단 각주를 소리 내어 읽는다. "유창하다고 사실인 건 아니다"까지가 한 세트. (오해 ①)
- **주의**: "AI가 이해한다/안다" 표현을 쓰지 않는다.

## 9장 · 컨텍스트

- **한 문장**: context window에는 한도가 있다 — 넘으면 구성에 따라 제거·요약되거나 요청이 실패할 수 있다.
- **우리 일로**: "수십 MB regression log 전체보다 fail 주변+summary만" — 토큰 요령이 이 장에 흡수됨(토큰 상세는 부록 45장).
- **풀어서**: "필요한 절차·자료만 골라 넣기 — Skill이 그 정리법" 복선.

## 10장 · 환각

- **한 문장**: 확률적 생성 과정에서 구조적으로 나타날 수 있는 실패 모드 — 유창함은 신뢰성과 다르다.
- **풀어서**: "거짓말"이라 부르지 않는다. 대응은 근거·도구·검증 절차로 감싸기(Harness 복선).
- **우리 일로**: 근거 없는 root cause 서술이 제일 위험 — 그래서 triage Skill에 "추정 금지"를 넣는다(20장 복선).

## 11장 · 무엇을 맡기나

- **한 문장**: AI가 잘하는 건 초안·분류·변환·요약 — 최종 판단은 사람.
- **우리 일로**: 좋은 일(regression 분류·spec 요약·review comment·register table) vs 조심할 일(sign-off·architecture·RTL 의미 변경·root cause 확정)을 실제 업무명으로 읽는다.

## 12장 · Agent 도입 4원칙

- **한 문장**: 작게 시작한다 · 출력을 구조화한다 · 근거를 남긴다 · 사람이 승인한다.
- **풀어서**: "오늘 이후 장들에서 이 배지가 보이면, 그 원칙이 작동하는 장면입니다" — 배지 규칙을 여기서 한 번만 설명.
- **주의**: 여기서 길게 설명하지 않는다 — 각 원칙은 해당 장면에서 다시 만난다.

## 14장 · 데이터 경계

- **한 문장**: 구분 기준은 접속 방법이 아니라 데이터의 처리·저장 위치다.
- **풀어서**: 오해 교정 두 개를 반드시 입으로 — "사내망에서 접속했다고 사내에서 처리되는 게 아니고, 브라우저로 쓴다고 무조건 외부인 것도 아닙니다." 애매하면 사내 경로 우선.
- **주의**: 이 슬라이드는 보안정책 선언이 아니다 — "구체 허용 범위는 회사 보안정책·승인된 서비스 기준" 각주를 언급. (오해 ③)

## 16장 · 사용 전 3문장 점검

- **한 문장**: 권한? 사외 반출? 검증? — 세 번 다 "예"일 때 도구를 켠다.
- **풀어서**: 체크 문구를 소리 내어 — "외부 메일로 보내도 되는가? 아니오라면 사외 AI에도 넣지 않습니다." 청중이 사진 찍어 가는 장.

## 18장 · 1단계 prompt — 두 공백

- **한 문장**: prompt만으로 데모 1까지는 간다 — 하지만 기준이 매번 다르고(절차), 시스템에 못 닿는다(연결).
- **풀어서**: 데모 1을 "1단계"로 소급 명명 — "우리는 이미 1단계를 봤습니다. 이제 한 단계씩 채웁니다."

## 19–20장 · 데모 2 (2단계 Skill)

- **풀어서**: "정의할 수 있으면 맡길 수 있다"가 축. 20장 셸에서 skill 자동 로드 줄과 unknown 3건 줄을 짚는다. 절차 5번("불확실하면 unknown — 추정하지 않는다") = 환각(10장)을 절차로 누르는 장치.
- **주의**: 구성된 예시 라벨. "매번 동일한 JSON"이 아니라 "같은 절차·같은 구조"로만.

## 21장 · Skill 개념

- **한 문장**: Skill은 매번 프롬프트를 길게 쓰지 않기 위한 팀 작업 매뉴얼.
- **주의**: 로드 방식은 클라이언트마다 다를 수 있다 — "자동으로 다 된다"처럼 말하지 않는다. 항상 적용되는 프로젝트 지침 / 필요할 때 로드되는 Skill / 연결하는 MCP 구분을 한 번.

## 22장 · Skill 구조·동작

- **한 문장**: 겉은 폴더 하나, 속은 frontmatter+절차 — 데모 2의 그 파일이 규격의 전부다.
- **풀어서**: progressive disclosure(평소엔 이름·설명만 → 관련 작업이면 본문 로드)를 컨텍스트 절약(9장)과 연결.

## 23–24장 · 데모 3 (3단계 MCP)

- **풀어서**: 같은 시나리오의 다음 질문 — "unknown 3건, 전에도 있었나?" 없을 때 = 연결이 사람(복붙 반복) / 있을 때 = jira_search 한 번. 매칭 없는 1건이 "신규 가능성 — 사람 확인"으로 남는 것까지 짚는다.
- **주의**: 자동화된 건 **조회·대조까지** — 원인 확정·티켓 갱신은 사람. 이슈 ID는 가상.

## 25장 · MCP 개념

- **한 문장**: MCP는 공용 연결 규격 — 손을 달아주는 것이지 판단력을 보장하는 것이 아니다.
- **주의**: "어디서나"가 아니라 "지원하는 클라이언트에서는". MCP는 보안 정책이 아니다 — 인증·권한은 별도 설계. (오해 ④)

## 26장 · MCP 통제

- **한 문장**: Agent는 권한을 가진 자동화 실행기 — 권한 설계가 곧 안전 설계.
- **풀어서**: prompt injection만 한 번 풀어 준다("문서·이슈 내용이 지시처럼 작동할 수 있다 — tool 결과는 불신 데이터로"). 마지막에 "두 공백이 채워졌다 — 남은 질문: 이 출력, 파이프라인에 넣어도 되나?"로 본문 3 브리지.

## 29–30장 · Harness란 / UVM 나란히

- **한 문장**: 이 세미나에서 Harness는 Agent 실행을 감싸는 검증·권한·재시도·로그·승인 구조 — verification harness와 같은 구조, 대상만 칩→모델.
- **주의**: "믿고 쓸 수 있게"가 아니라 "통제하게" — 양쪽 다 최종 판단은 사람.

## 31장 · Harness 구성 요소

- **한 문장**: 스키마 하나가 아니라 통제 장치들의 묶음 — 믿기 위한 장치가 아니라 통제하기 위한 장치. (오해 ⑥)
- **풀어서**: v1.1 추가 항목을 짚는다 — evidence·confidence 필드(근거·불확실성을 출력에), read-only 기본·dry-run(쓰기 전 diff는 사람이).

## 32장 · 스키마 게이트

- **한 문장**: 구조가 맞다 ≠ 판단이 맞다 · 실행이 끝났다 ≠ 업무가 완료됐다.
- **우리 일로**: "assertion이 violation을 잡아도 spec 해석은 별도 리뷰인 것과 같다." (오해 ⑤)

## 33장 · 언제 쓰나

- **풀어서**: 5질문 체크를 그대로 읽는다 — 자동 입력되나 · 영향 크나 · 되돌리기 어렵나 · 반복되나 · 쓰기 권한 쓰나. 하나라도 "예"면 Harness.

## 34장 · 자율성 단계 L0–L5

- **한 문장**: 처음부터 L5로 가면 사고도 자동화된다 — 시작은 L1~L2.
- **주의**: **외부 표준이 아니라 이 세미나의 실무용 예시**임을 말한다(슬라이드에도 명시됨). 데모 1=L1, 2=L2, 3=L3, 4=L4.

## 35장 · 데모 4 (4단계 Harness)

- **주의**: 대조 축은 품질+재현성 — 재현성은 "같은 검증 절차·같은 출력 구조 통과"이지 같은 답이 아님. 성능 수치·퍼센트 금지. bad run 3종은 벤더 문서가 인정하는 실제 현상.

## 36장 · 회고

- **풀어서**: "오늘 우리는 데모 1의 작업 하나를 실무 시스템 직전까지 키웠습니다" — 단계 여정을 한 호흡으로 복기 + 4원칙 콜백.

## 37장 · 좋은/나쁜 요청

- **한 문장**: 작게 · 구체적으로 · 검증 가능하게. 좋은 요청 3개는 청중이 복사해 갈 문장 — 그대로 읽는다.

## 38장 · 설계 템플릿 9칸

- **한 문장**: 코드가 아니라 글 아홉 줄 — 이게 Skill 초안이고, 승인·실패 줄이 Harness 요구사항이다.
- **풀어서**: "각자 업무 하나를 떠올리고, 이 아홉 칸을 채워 보세요"가 실습 유도 멘트. 오른쪽 triage 예시를 한 줄씩 짧게.

## 39장 · 첫 실험 CTA

- **한 문장**: 첫 목표는 자동화가 아니라, 내가 검증 가능한 반복 작업 하나를 줄이는 것.
- **풀어서**: 5단계 실험 절차(하나 고르기→입력 3개→비교→오류 기록→결정)를 "내일 할 일"로 읽는다. 지표 4개(수정 횟수·누락·오분류·근거 없는 주장)면 충분.
