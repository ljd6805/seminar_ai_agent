# AI Agent 기술 세미나 — 발표 시놉 (v0.8)

## 기본 정보

- **목표**: AI agent를 처음 소개하고, 팀원들이 "나도 써봐야겠다"는 동기를 갖게 하기
- **청자**: 반도체 설계/검증 담당자. AI agent 이해도 낮음
- **핵심 서사**: *AI agent가 있다 → 이런 장점이 있다 → 이렇게 쓰면 된다* (하나의 큰 맥락으로 소개)
- **업무환경**:
  - 사내 서버 LLM 서빙 (API 형태 · OpenCode 형태)
  - 사외 Claude Code (도입 예정)
- **자료형태**: 가로 슬라이드 HTML / 키보드 네비게이션 / 애니메이션·인터랙션

---

## 전체 구조 (한눈에)

```
도입    AI agent란 게 있다
본문
  └ 우리 환경부터        → 본문 1 : 사내/사외 정보 흐름
  └ 부품을 이해한다      → 본문 2 : plugin / skill / mcp
  └ 틀로 감싸 제대로 쓴다 → 본문 3 : harness
마무리  (보류)
```

본문은 **우리 환경 → 부품 → 틀** 순으로, 뒤로 갈수록 난이도가 우상향하는 상승 구조.

## 도입 — "AI agent란 게 있다"

1. **AI의 시작·역사**
   - 오늘 다룰 'agent'가 어디서 왔는지 감 잡기
   - **6개 구체 마일스톤(연도·명칭, 공식 출처 검증)**: 2012 AlexNet → 2017 Transformer → 2020 GPT-3(few-shot) → 2022 ChatGPT(instruction tuning·RLHF) → 2023 function calling → 2024–25 MCP·agent
   - 각 단계가 다음 단계의 발판(상승 서사). 지루해지기 쉬운 구간 → 시각 효과로 리듬 있게

2. **AI agent의 정의 + 패러다임 전환** ⭐
   - 기존 업무 방식과 뭐가 다른가
   - **여기에 오프닝 데모(Ex1)를 실증으로 배치**
     - *"이 발표자료도 AI로 만들었다"* — 이 슬라이드를 만든 대화 과정 자체를 애니메이션으로
     - 형식: 왼쪽에 대화가 오가고, 오른쪽에서 슬라이드 얼개가 한 줄씩 자라나는 그림 (A안)
     - 재생: 자동 재생, 시선을 끄는 수준
     - 역할: 단순 훅이 아니라 **"일하는 방식이 바뀐다"는 개념의 증명**
     - ※ 구체적 표현/디자인은 추후

3. **왜 지금 우리가**
   - 바뀔 때 vs 안 바뀔 때, 변화의 필요성 어필

## 본문 1 — 사내/사외 정보 흐름 : "우리 환경부터"

- 사내 서버 LLM (API · OpenCode) vs 사외 Claude Code (도입 예정)
- **정보가 사내에 머무는가, 사외로 나가는가**를 시각화
- 보안 엄격한 환경이라 팀원들이 가장 먼저 궁금해할 지점 → 본문의 문을 여는 자리
- 이후 나올 부품·harness 얘기의 전제 배경이 됨

---

## 본문 2 — plugin / skill / mcp : "부품을 이해한다"

1. **개념·동작원리** (2장: 정의 / 동작+관계)
   - skill = 모델이 호출하는 능력(SKILL.md) · mcp = 외부 연결 표준 프로토콜(client–server, tools·resources·prompts) · plugin = 확장 배포 패키지(marketplace)
   - **전문 용어 + 각 용어를 쉬운 정의로 받쳐줌** (레고·부품상자 비유는 폐기). 세 개는 같은 층이 아니라 배포/능력/연결의 서로 다른 축임을 관계도로 명시

2. **간단한 동작 예시 (Ex2)**
   - agent와 skill이 실제로 이렇게 움직인다, 눈으로 바로 확인
   - 개념을 들은 직후 체감시키는 자리

3. **유용한 것들 소개 + 설치법 + 주의사항**
   - 대표 skill · mcp · plugin — **1장에 압축**. 3칸(Context확장 / Web / Token절약) × 각 대표 1개.
     - 카테고리 지도(어떤 결의 도구들이 있나)만 본문에 전달, 도구명 나열은 지양
     - 전체 후보 목록은 **부록 B**로 (발표 시간 비용 0, 세미나 후 자가학습 경로)
       - Context 확장: ralph loop, claude-mem, superpowers
       - Web 개발: firecrawl, frontend design, playwright
       - Token 절약: caveman, ponytail, yagni
   - 설치 방법
   - 보안 주의사항 — 별도 섹션 없이 여기서 **간단히만** 언급

---

## 본문 3 — harness : "틀로 감싸서 제대로 쓴다"

1. **harness란 + 작동방식**
   - 모델을 감싸 순서 지정 + 출력 검증 + 실패 시 재시도로 돌리는 실행 틀
   - **청중 언어로 재구성**: DUT를 감싸 정해진 자극·검사를 가하는 **verification harness(UVM testbench)와 같은 결**, 대상이 모델일 뿐. 스키마 검증 = assertion·checker 게이트

2. **장점과 단점**
   - 뭘 얻고 뭘 치르는지 균형 있게

3. **harness 유무 성능 차이 (Ex4)** — 클라이맥스
   - 있을 때 vs 없을 때

## 마무리 — "변화 재강조 → 안전한 첫걸음" (순서 결합)

두 방향(① "일하는 방식의 변화" 재강조 / ② "뭘 더 해볼까" 액션)을 택일하지 않고 **순서로 이어 붙인다.**
먼저 큰 메시지로 감정을 한 번 데우고(정점), 곧바로 사내 경계 안 안전한 첫 작업 하나로 착지시킨다.
마지막 한 문장에서 둘을 융합.

- 이유: "변화" 재강조만으로 닫으면 "그래서 뭘?"이 안 생겨 세미나 목표(직접 써보게 만들기)에 못 닿고,
  "액션"만으로 닫으면 밋밋하고 '숙제 주는' 압박이 남는다. 둘을 이으면 서로의 약점을 메운다.

1. **다시 그 장면 (정서 정점)**
   - 오프닝 Ex1을 **정지컷 1장**으로 되짚음 (재생 X). "처음엔 신기하기만 했는데 이제 원리가 읽히죠?"
   - 경탄 → 자신감으로 전환. 구경거리였던 것이 '내가 아는 것'이 됨

2. **그래서 월요일에 뭐부터? (핵심 CTA)**
   - 4기준 나열해 "스스로 고르라" 하지 않고 **딱 하나 집어줌**
   - 사내 LLM(API·OpenCode)에서 · 데이터가 사내를 벗어나지 않는 갈래에서(본문 1 콜백) ·
     설계/검증 맥락의 작고 눈으로 검증되는 작업(예: 스펙 문서 구조화, 로그 요약) · **사람이 최종 검수하는 루프** 유지
   - "애매하면 보안 먼저 확인" 안전판 명시

3. **질문 받습니다 (가볍게 + Q&A)**
   - 강제 공유·옆사람 발표 같은 부담 장치 없음. 지지대(부록 B 자가학습·옆자리 동료) 한 줄
   - 마지막 한 마디로 청중을 '개념 이해'에서 '이번 주 개인 행동'으로 밀어냄

> **마지막 한 마디(안):** "바뀐 건 도구가 아니라, 그 장면을 읽어내는 여러분입니다 —
> 이번 주, 사내에서 딱 하나만 켜서 그 첫 대화를 시작해 보세요."

**지켜야 할 것 (심사에서 걸린 지점)**
- Ex1은 마무리에서 **딱 한 번만** 소환(정점에서만). 두 번 나오면 "또 그 얘기"로 지겨워짐
- 즉시 CTA가 **Ex4와 충돌하지 않게** — Ex4가 "무-harness 단발 호출은 흔들린다"고 했으니,
  즉시 행동은 '사람이 검수 루프'로 봉합("완벽한 틀은 도입 후, 지금은 사람이 그 자리를")
- 예시는 회의록 같은 범용 말고 **설계/검증 실무**(스펙·로그)로 → 남 얘기처럼 안 들리게
- 사외 Claude Code는 '도입 예정'이라 즉시 CTA에서 빼고 **'도입되면'으로 시제 분리**

> ※ 세부(정점 15초 길이, 참여 요소 유무 등)와 시각 표현은 슬라이드 제작 단계에서 조정. 지금은 방향·골격만 확정.

## 확정 / 미결 / 보류 정리

**확정된 것**
- 핵심 서사: agent 소개 → 장점 → 활용법, 하나의 큰 맥락
- 본문 세 파트: 사내/사외 정보 흐름 → plugin/skill/mcp → harness
- **사내/사외 정보 흐름을 본문의 첫 섹션(본문 1)으로 배치**
- **본문 2 내부 순서: 개념 → 예시 → 소개·설치**
- 오프닝 데모(Ex1)를 도입 2번(정의·패러다임)으로 이동, 개념의 실증으로 승격
- Ex1 애니메이션은 A안(대화 재생 + 얼개 성장), 자동 재생
- GitOps 협업(Ex3) 제외
- 설치 주의사항은 본문 2에 흡수, 간단히만
- **본문 2 "유용한 것들"은 1장(카테고리 3칸 × 대표 1개), 전체 목록은 부록 B**
- **마무리는 "변화 재강조 → 안전한 첫걸음" 순서 결합, 3장 골격**(정점 → CTA → Q&A)
- **(v0.8) 콘텐츠 전문화**: 역사 6마일스톤 구체화, skill/mcp/plugin 정확 정의+동작+관계, harness를 verification harness(UVM)에 빗대는 재구성. 5갈래 조사+검증(공식 문서 대조)로 근거 확보. 성능 수치는 계속 미사용
