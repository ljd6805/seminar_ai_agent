# AI Agent 기술 세미나 — 발표 시놉 (v2.5)

## 기본 정보

- **목표**: 청중이 ① LLM을 대략 이해하고 ② AI Agent가 무엇인지 알고 ③ Skill/MCP/Harness의 역할을 구분하고 ④ 세미나 후 **자신이 바로 검증 가능한 작은 업무 하나**에 Agent를 적용해 보게 하기
- **청자**: 반도체 설계/검증/소프트웨어 엔지니어. AI Agent 처음이거나 ChatGPT 정도만 경험
- **발표 시간**: **90분**
- **핵심 메시지(고정)**: **"AI Agent는 사람의 판단을 대체하는 도구가 아니라, 판단 전에 필요한 반복 작업을 줄이는 도구다."**
- **관통 시나리오(v1.8)**: **Regression Triage Agent** — 하나의 작업(regression 실패 분류)을 데모 1(prompt만) → 데모 2(Skill) → 데모 3(MCP) → 데모 4(Harness) → 사람 승인으로 단계적으로 키운다. 회의 메모 등 다른 예시는 보조·부록.
- **4요소(v2.3.3, 3장 Master Visual 안에 한 문장 정의)**: Agent(목표를 향해 도구를 사용하며 작업을 이어가는 **실행 시스템**) / Skill(반복 업무의 절차와 기준을 담은 **작업 매뉴얼**) / MCP(Agent와 데이터·도구를 연결하는 **공용 규격**) / Harness(권한·검증·로그·승인으로 실행을 통제하는 **구조**). 종류 구분(실행 시스템/작업 매뉴얼/공용 규격/통제 구조)이 읽히는 것이 목표 — 상세 설명은 이후 장.
- **Agent 도입 4원칙(v1.8, 12장에서 1회 정의 후 배지로 반복)**: ① 작게 시작한다 ② 출력을 구조화한다 ③ 근거를 남긴다 ④ 사람이 승인한다
- **(v2.3.3, 2026-07-16) 3장 4요소 정의 보강**: 네 노드의 짧은 키워드를 한 문장 정의로 교체(Skill·MCP 구 키워드는 정의에 포함돼 삭제, Agent 구 키워드 "LLM 기반 · 계획 · 도구 사용 · 확인 ↻"는 약한 보조 문구로 강등) · Harness 헤더 = 정의, 범위 pill 3개 → 1개("Agent 실행 전 · 중 · 후 전체에 적용" — 구 헤더 문구 이동) · 후반 콜백 일치(23장 lead·용어집 MCP = "Agent와 데이터·도구를 연결하는 공용 규격", 노트 19장 "팀 작업 매뉴얼" → "작업 매뉴얼"). 장 수·번호·발표 모드 불변
- **(v2.3, 2026-07-13) 8·9장 중복 제거 + 8장 WHY 복원**: 8장 = "반복에 묶인 시간을, 판단으로 옮기기 위해서입니다"(왜 쓰나 전용 — 상단 레일 + 시간의 이동 2행, 하단 협업 레일 삭제, note = 3장 핵심 메시지 콜백) · 9장 피드백 점선 삭제(수정 방법은 10장 소유) → 중간 개입 한 줄 · 그림 소유 정리(레일=8 / 구조=9 / 루프=10). 장 수·번호 불변
