# AI Agent 기술 세미나 — 발표 시놉 (v1.7)

## 기본 정보

- **목표**: 청중이 ① LLM을 대략 이해하고 ② AI agent가 무엇인지 알고 ③ skill/MCP/harness의 역할을 구분하고 ④ 세미나 후 **자신이 바로 검증 가능한 작은 업무 하나**에 agent를 적용해 보게 하기
- **청자**: 반도체 설계/검증/소프트웨어 엔지니어. AI agent 처음이거나 ChatGPT 정도만 경험
- **발표 시간**: **90분**
- **핵심 메시지(고정, v1.7)**: **"AI agent는 사람의 판단을 대체하는 도구가 아니라, 판단 전에 필요한 반복 작업을 줄이는 도구다."**
  - 관통 4문장: ① agent = LLM에 도구와 반복 루프를 붙인 실행자 ② skill = 일하는 방법을 알려준다 ③ MCP = 일할 시스템에 연결한다 ④ harness = 결과가 다음 단계로 넘어가도 깨지지 않게 감싼다
- **핵심 서사(척추)**: *"알고, 써보자"* — 3장에서 결론을 먼저 선언하고, 본문이 네 문장을 하나씩 채운다
  - **(v0.9 확정)** "일하는 방식을 바꿔야 한다"(변화 설득)는 척추가 아님 — **클로징 미래형 한 줄로만**
- **톤(v1.7)**: "알아서 다 해준다" 금지 — **"정해진 권한과 한도 안에서 반복 작업을 도와준다"**. 피하는 표현(무조건·완벽하게·어디서나·항상·믿고 맡기면·될 때까지 돈다·표준 방법) → 선호 표현(정해진 범위 안에서·검증 가능한 작은 작업부터·권한과 한도를 두고·사람이 최종 판단·재사용 가능성이 커짐·지원하는 환경에서는)
- **업무환경**: 사내 직접 서빙(OpenCode · Roo · DS assistant API — 데이터 사내 유지) / 사외 Claude(도입 예정 — 외부 전송)
- **자료형태**: reveal.js 단일 파일 HTML(vibe 테마) / autoshow(진입 시 자동 표시, fragment 없음) — **53장** (발표 51장 + 부록 2장)
- **발표 대본**: [speaker-notes.md](speaker-notes.md) — 장별 핵심 문장·설계/검증 연결·오해 바로잡기 6·90분 배분

## 전체 구조 (한눈에, v1.7 = 53장)

```
도입    결론 먼저, 그리고 기초   → 1 표지 · 2 목차 · 3 오늘의 결론(핵심 메시지+4문장) · 4 역사 · 5–8 기초(LLM·컨텍스트·토큰·환각)
                                  · 9 데모1(Ex1) · 10 해부 · 11 정의(권한·한도) · 12 챗봇 vs agent · 13 왜 지금 · 14 왜 우리
                                  · 15 일의 재배치 · 16 무엇을 맡기나(초안·분류·변환·요약)
본문 1  우리 환경부터            → 17 간지 · 18 격리 다이어그램(+권한) · 19 사내 도구 셋 · 20 보내도 되는 것·안 되는 것(체크 문구)
                                  · 21 사용 전 3문장 점검 · 22 OpenCode 설치
본문 2  확장 요소를 이해한다     → 23 간지 · 24 기본 모델의 한계(두 공백) · 25–26 데모2(업무 정의→skill 저장·실행)
                                  · 27–28 skill(개념/구조) · 29 triage skill(설계·검증 실전) · 30 좋은/나쁜 skill
                                  · 31–32 데모3(MCP 없을 때/있을 때) · 33–34 MCP(개념/구조) · 35 MCP 통제 · 36 도구 지도
본문 3  틀로 통제해 제대로 쓴다  → 37 간지 · 38 실패의 현실 · 39 harness란 · 40 UVM 나란히 · 41 harness 구성 요소
                                  · 42 스키마 게이트(모양≠의미) · 43 장단점(재현성 정의) · 44 언제 쓰나 · 45 자동화 레벨 L0~L5 · 46 데모4
마무리  회고 → 요청법 → 첫걸음   → 47 회고 · 48 FAQ · 49 좋은/나쁜 요청 · 50 첫걸음 CTA(+첫 프롬프트 2종) · 51 Q&A 클로징
부록    (발표 시 넘기지 않음)    → 52 도구 후보 목록 · 53 용어집
```

본문은 **우리 환경 → 확장 요소 → 틀** 순으로 난이도 우상향. 각 섹션은 "그래서 실무에서 뭐가 좋은가"가 보이게(데모 선행 + 도메인 예시).

### 90분 배분 (v1.7)

문제 제기·결론(1–3, ~5분) → LLM 기초(4–8, ~10분) → agent 정의+데모 1(9–16, ~15분) → 보안·환경(17–22, ~10분) → skill(23–30, ~15분) → MCP(31–36, ~15분) → harness(37–46, ~12분) → 마무리·Q&A(47–51, ~8분)

## 도입 (1~16장, 약 30분)

1. **표지·목차** (1–2장)
2. **오늘의 결론** (3장, v1.7 신설) — 처음 3분 안에 결론: 핵심 메시지 인용 박스 + 쉬운 업무 예 3개(regression 로그 분류·회의록 action item·spec 변경점 요약) + 관통 4문장. "목표는 완벽한 이해가 아니라 이번 주 작은 업무 하나 적용"
3. **역사** (4장) — 7노드, 짧게(1.5분). v1.7 보수화: AlexNet "실용성을 증명한 상징적 사건" · function calling "대표 API 패턴" · MCP "공용 규격으로 확산 중"
4. **기초 4장** (5–8장) — 진입 시 1회 재생 + ↻:
   - **LLM** (5장) — 다음 말 예측 + **"엄밀히는 다음 토큰" 각주**. 유창함 ≠ 사실 — 근거 확인·검증 필요
   - **컨텍스트** (6장) — 한도 초과 시 잘림/요약·길수록 집어내기 어려움(사람 기억 비유 완화). skill = 컨텍스트 절약 정리법 복선
   - **토큰** (7장) — kWh 비유 + 실무 요령("수십 MB log 전체보다 fail 주변+summary")
   - **환각** (8장) — "모른다고 멈추기보다 그럴듯한 답으로 기울 수 있다" + **"유창함은 신뢰성과 다릅니다"** + 근거·도구·검증 절차로 감싸기(harness 복선)
5. **데모 1 (Ex1)** (9장) ⭐ — regression 로그 실패 분류(구성된 예시, 1회 재생+↻)
6. **해부 → 정의 → 대조** (10–12장) — 정의(11장)는 **"정해진 권한과 한도 안에서 이해→계획→실행→확인 반복"**. 챗봇 대조(12장)는 공정하게("묻고 답하기엔 챗봇으로 충분")
7. **왜 지금 → 왜 우리 → 재배치 → 위임 기준** (13–16장) — 16장 = "초안·분류·변환·요약은 AI, 최종 판단은 사람" + 도메인 예시 표

## 본문 3 — harness (37~46장, 약 12분)

1. **간지** (37장) — **"형식과 실행을 통제하는 틀"**(v1.7, "믿고 쓸 수 있게" 폐기)
2. **실패의 현실** (38장) — 환각 콜백 · 형식 변동
3. **harness란 / UVM** (39–40장) — 공식 근거 "agentic harness". UVM 병렬 + **양쪽 다 최종 판단은 사람**(spec 해석은 엔지니어가 / 의미·업무 판단은 사람이)
4. **구성 요소** (41장, v1.7 신설) — 입력(전처리·skill 선택·tool allowlist)/출력(schema·validator·retry limit)/운영(human approval gate·로그 tracing·실패 리포트) — **"믿기 위한 장치가 아니라 통제하기 위한 장치"**
5. **스키마 게이트** (42장) — 장점(누락 검출·파싱 실패 감소·retry 조건) + 한계 — **"스키마는 모양을 검증합니다. 의미는 별도로 검증해야 합니다"** + assertion↔spec 리뷰 비유
6. **장단점 → 언제 쓰나 → 레벨** (43–45장) — 재현성 = **"매번 같은 답이 아니라 같은 검증 절차·같은 출력 구조 통과"** / "사람에게 가면 agent, 프로그램에게 가면 harness" / **L0~L5 사다리**(v1.7 신설 — "처음부터 L5면 사고도 자동화, 시작은 L1~L2")
7. **데모 4 (Ex4)** (46장) — harness 유무 3회 대조, "같은 구조"까지만
