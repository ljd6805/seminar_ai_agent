# AI Agent 기술 세미나 — 발표 시놉 (v1.6)

## 기본 정보

- **목표**: AI agent를 처음 소개하고, 팀원들이 "나도 써봐야겠다"는 동기를 갖게 하기
- **청자**: 반도체 설계/검증 담당자. AI agent 이해도 낮음
- **발표 시간**: **90분**
- **핵심 서사(척추)**: *"알고, 써보자"* — AI agent가 무엇이고 → 우리 환경에서 어떻게 쓰고 → 사내에서 첫걸음까지
  - **(v0.9 확정)** "일하는 방식을 바꿔야 한다"(변화 설득)는 척추가 아님 — 주장하지 않고 청중이 결론내리게 하며, **클로징 미래형 한 줄로만** 남김. 표지 제목도 A 행동 선언("오늘부터 쓸 수 있습니다").
- **업무환경**:
  - 사내 직접 서빙 서비스: OpenCode · Roo · DS assistant API — 데이터가 회사 경계를 벗어나지 않음
  - 사외 Claude (도입 예정) — 입력이 외부 서버로 전송
- **자료형태**: reveal.js 단일 파일 HTML(vibe 테마) / 키보드 네비게이션 / autoshow(진입 시 자동 표시, fragment 없음) — **45장** (발표 43장 + 부록 2장)

본문 2  확장 요소를 이해한다    → 21 간지 · 22 기본 모델의 한계(두 공백) · 23–24 데모2(skill: 업무 정의→저장·실행) · 25–26 skill(개념/구조·동작)
                                  · 27–28 데모3(MCP 없을 때/있을 때) · 29–30 MCP(개념/구조·동작) · 31 도구 지도+설치·주의
본문 3  틀로 감싸 제대로 쓴다   → 32 간지 · 33 실패의 현실 · 34 harness란 · 35 UVM 나란히 · 36 스키마 게이트
                                  · 37 장단점 · 38 언제 쓰나 · 39 데모4(harness 유무)
마무리  회고 → 첫걸음 → Q&A    → 40 오늘 지나온 길(회고) · 41 FAQ · 42 첫걸음 CTA(+첫 프롬프트) · 43 Q&A 클로징
부록    (발표 시 넘기지 않음)   → 44 도구 후보 목록 · 45 용어집

본문은 **우리 환경 → 확장 요소 → 틀** 순으로 난이도 우상향. 본문 1~3은 간지로 시작.
(※ **v1.3: plugin은 발표에서 제외** — 확장 요소 = skill·MCP 둘. 본문 2는 요소마다 "개념 → 구조·동작 → 데모"로 완결. MCP는 약어라 대문자.)

### 확장의 설계 원리 (v1.0, 2026-07-07)

- **A 서사 구멍 해소**: 문제→해결 순서(맨몸 한계 먼저) · 신뢰 캘리브레이션 조기화(실패의 현실) · 용어 선행 정의(LLM 기초) · 데모의 중간 고리(SKILL.md 실물) · CTA 구체화(첫 프롬프트)
- **B 압축 해체**: skill·MCP 각 2장(개념/구조·동작) — 입문 페이스 확보 (역사는 v1.2에서 1장 재병합)
- **C 도메인 공명**: 설계·검증의 반복 pain, UVM 병렬 다이어그램, 사내 도구·판단 사례
- "많이 만들고 줄이기" — 이후 리허설에서 압축 시 페이지리스트에서 삭제 이력 관리

## 도입 — "AI agent란 게 있다" (1~15장, 약 30분)

1. **표지·목차** (1–2장) — 목차는 5부 여정 + "낯선 단어는 그 자리에서 풀어서 + 부록 용어집" 안심 문구
2. **역사** (3장) — 7노드 타임라인(2012 AlexNet → 2024–25 agent loop). 두 시대("말하는 AI"/"일하는 AI") 구분은 lead, "불과 2~3년" 복선은 note
3. **기초 4장** (4–7장, v1.4에서 1장→4장 분리) — 네 장 모두 리드 + 애니메이션 패널 + note 브리지. **진입 시 1회 재생 후 정지**(설명용) + "↻ 다시 재생" 버튼:
   - **LLM** (4장) — 다음 말 예측 애니메이션(문장+확률 후보 바→선택→이어 붙임, CSS 루프). agent와의 차이는 정의(10장)로 이동
   - **컨텍스트** (5장) — 차오르는 창 애니메이션(칩 6개 → 게이지 한도 → 앞부터 흐려짐) → "관리가 곧 실력", skill의 필요할 때만 로드 복선
   - **토큰** (6장) — 간결 유지(kWh 비유). 문장 조각내기 애니메이션(토큰 하이라이트 6단계 + 사용량 미터). 컨텍스트 한도도 토큰으로 셈
   - **환각** (7장) — 대화 3박자 애니메이션(근거 없는 질문 → 유창한 오답+"지어낸 답" 스탬프 → 근거를 주면) → "알고 쓰자" + harness(본문 3) 복선
4. **데모 1 (Ex1)** (8장) ⭐ — show→tell. *"regression 로그 실패 분류"* 장면(구성된 예시 라벨) — 대화 6버블 → 리포트 5행 **1회 시퀀스 재생 + ↻ 버튼**
5. **데모 1 해부** (9장) — 세 장면을 핵심 3요소로 매핑
6. **정의** (10장) — "방금 보신 것처럼—" 콜백 + **LLM+도구+루프=AI agent 등식·순환 레인 애니메이션**(1회 재생 → 확인 점등·agent 글로우로 정지 + ↻ 버튼). "도구 사용 · 다단계 진행 · 결과 확인"
7. **챗봇 vs agent** (11장) — 같은 LLM, 차이는 도구+루프
8. **왜 지금 → 왜 우리 → 재배치 → 위임 기준** (12–15장) — 일반 근거 → 도메인 pain → 역할 재배치(기술 톤, 척추 A) → "결과를 내가 바로 확인할 수 있는가" + 본문 1 브리지

## 본문 2 — 확장 요소: skill · MCP (21~31장, 약 28분) — v1.3: plugin 제외, "개념→구조·동작→데모" 배열

1. **기본 모델의 한계** (22장) — **두 공백**: 절차(우리 절차 모름)·연결(시스템에 못 닿음) — 카드마다 "이 공백을 채우는 확장 요소" skill/MCP 대칭 배너로 대응 시각화 → "절차는 skill, 연결은 MCP"
2. **skill** (23–26장, show→tell) — **데모 2 먼저**: ① 업무 정의(사람의 일 3단계 + 정의 박스, 23장) → ② skill 저장(`.opencode/skills/meeting-notes/SKILL.md`)·실행(opencode 프롬프트 한 줄 → skill 자동 로드 → JSON, 20장 셸 콜백)·활용(주간 리포트 조립, 24장) → 개념(없으면/있으면 + 매뉴얼 비유, 25장) → 구조·동작(OpenCode 규격 파일트리 + 만드는 법 세 걸음 + SKILL.md 해부(YAML frontmatter/Markdown 본문 존) + progressive disclosure flow, 26장)
3. **MCP** (27–30장, show→tell) — **데모 3 먼저 (Ex5, 2장)**: 없을 때(연결=사람 다이어그램, 수동 복붙 — "가져오기를 맡길 수는 없을까?", 27장) / 있을 때(연결=MCP 서버, tool 호출 jira_search→상태별 정리, 구성된 예시 라벨, 28장) → 개념(N×N → USB-C, OpenAI·Google 채택, 29장) → 구조·동작(client–server + opencode.json 연결 설정 실물(jira, 주소 가상) + 동작 flow ①②③ = "데모 3의 그 순서", "두 공백 모두 채워짐" 마무리, 30장)
4. **도구 지도 + 설치·주의** (31장) — 3칸 지도 × 예 1개, 설치 한 줄, 주의 한 줄(일부 MCP 서버·도구는 외부 연결 → 보안 콜백)
