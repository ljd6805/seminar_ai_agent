# AI Agent 기술 세미나 — 발표 시놉 (v2.3)

## 기본 정보

- **목표**: 청중이 ① LLM을 대략 이해하고 ② AI Agent가 무엇인지 알고 ③ Skill/MCP/Harness의 역할을 구분하고 ④ 세미나 후 **자신이 바로 검증 가능한 작은 업무 하나**에 Agent를 적용해 보게 하기
- **청자**: 반도체 설계/검증/소프트웨어 엔지니어. AI Agent 처음이거나 ChatGPT 정도만 경험
- **발표 시간**: **90분**
- **핵심 메시지(고정)**: **"AI Agent는 사람의 판단을 대체하는 도구가 아니라, 판단 전에 필요한 반복 작업을 줄이는 도구다."**
- **관통 시나리오(v1.8)**: **Regression Triage Agent** — 하나의 작업(regression 실패 분류)을 데모 1(prompt만) → 데모 2(Skill) → 데모 3(MCP) → 데모 4(Harness) → 사람 승인으로 단계적으로 키운다. 회의 메모 등 다른 예시는 보조·부록.
- **4요소(v1.8, 3장에서 동등 4행으로 시각화 — 트리 폐기)**: Agent(생각하고 실행 — LLM+도구+루프) / Skill(일하는 방법·규칙) / MCP(일할 데이터·도구 연결) / Harness(AI가 실수해도 안전하게 — 검증·권한·로그·승인). 질문 4 — 어떻게 움직이나 · 무엇으로 일하나 · 무엇과 연결되나 · 어떻게 통제하나.
- **Agent 도입 4원칙(v1.8, 12장에서 1회 정의 후 배지로 반복)**: ① 작게 시작한다 ② 출력을 구조화한다 ③ 근거를 남긴다 ④ 사람이 승인한다
- **(v2.3, 2026-07-13) 8·9장 중복 제거 + 8장 WHY 복원**: 8장 = "반복에 묶인 시간을, 판단으로 옮기기 위해서입니다"(왜 쓰나 전용 — 상단 레일 + 시간의 이동 2행, 하단 협업 레일 삭제, note = 3장 핵심 메시지 콜백) · 9장 피드백 점선 삭제(수정 방법은 10장 소유) → 중간 개입 한 줄 · 그림 소유 정리(레일=8 / 구조=9 / 루프=10). 장 수·번호 불변
- **(v2.2, 2026-07-13) 10장 검증 루프 순환 재설계**: 리본 일렬 → 닫힌 2×2 루프(①작게 시작→②출력 구조화→③근거 확인→④사람 승인) + 승인 분기(점선 "수정 필요"=같은 범위 재실행 ④→② / 앰버 실선 "승인 후 다음 작은 범위" ④→①) + 중앙 "한 바퀴씩 확인하며 확대" — 반복의 재시작 조건 = 사람의 판단(무한 자동 실행 아님). 장 수·번호 불변
- **(v2.1, 2026-07-13) 7~10장 업무 방식 서사(WHAT→WHY→WHO→OPERATE)**: 새 8장 "Agent는 업무의 중간 실행 단계를 나눠 맡을 수 있습니다"(기존 6단계 사람 레일 vs 사람→Agent→사람 협업 레일 — 더 똑똑해서가 아니라 도구·실행 루프 때문, 사람은 중간에도 개입) 신설 · 구 8장(무엇을 맡기나) → 9장 "역할 분담"(사람: 목표·기준·범위 / Agent: 탐색·분류·변환·초안 / 사람: 근거 확인·수정·판단·승인 + "수정 요청" 피드백 점선, 좋은 일·조심할 일은 보조 스트립으로) · 구 9장(4원칙) → 10장 "작은 검증 루프"(리본 유지 + ↺ 유지·수정·확대 회귀 화살표, "안전 보장 공식이 아니라 시작 조건") · 부록 일의 재배치(새 46장) = 책임 보충으로 재구성(본편 반복 방지) · **총 55장 = 본편 38 + 부록 17, 발표 모드 35장**, 구 8장 이후 번호 +1
- **(v2.0, 2026-07-13) 7장 재설계 + LLM 기초 부록 이동**: 7장 = "LLM은 모델, AI Agent는 실행 시스템"(챗봇 대조 폐기 — 챗봇은 인터페이스·제품 형태, 좌우 동일 LLM 노드 + Agent 컨테이너가 LLM·허용된 도구·실행 루프를 포함, 같은 요청 대조, 하단 = 구조 차이 + "사실성 자동 보장 아님") · 구 8·9·10장(LLM·컨텍스트·환각)을 부록 38~41 "LLM 기초 ①~④"로 이동(45 토큰을 ③으로 재배치 — 다음 토큰→컨텍스트→토큰→환각) · 본편 37 + 부록 17(총 54 유지), 구 11장 이후 본편 번호 -3, 발표 모드 34장 · 새 19장(Skill 구조)에 컨텍스트 유한성 한 줄 · 본편 "환각" 첫 등장(25장)을 풀어 쓴 설명과 함께
