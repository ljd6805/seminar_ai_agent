# AI Agent 기술 세미나 — 발표 시놉 (v0.6)

## 기본 정보

- **목표**: AI agent를 처음 소개하고, 팀원들이 "나도 써봐야겠다"는 동기를 갖게 하기
- **청자**: 반도체 설계/검증 담당자. AI agent 이해도 낮음
- **핵심 서사**: *AI agent가 있다 → 이런 장점이 있다 → 이렇게 쓰면 된다* (하나의 큰 맥락으로 소개)
- **업무환경**:
  - 사내 서버 LLM 서빙 (API 형태 · OpenCode 형태)
  - 사외 Claude Code (도입 예정)
- **자료형태**: 가로 슬라이드 HTML / 키보드 네비게이션 / 애니메이션·인터랙션

---

## 전체 구조 (한눈에)

```
도입    AI agent란 게 있다
본문
  └ 우리 환경부터        → 본문 1 : 사내/사외 정보 흐름
  └ 부품을 이해한다      → 본문 2 : plugin / skill / mcp
  └ 틀로 감싸 제대로 쓴다 → 본문 3 : harness
마무리  (보류)
```

본문은 **우리 환경 → 부품 → 틀** 순으로, 뒤로 갈수록 난이도가 우상향하는 상승 구조.

## 도입 — "AI agent란 게 있다"

1. **AI의 시작·역사**
   - 오늘 다룰 'agent'가 어디서 왔는지 감 잡기
   - 지루해지기 쉬운 구간 → 시각 효과로 리듬 있게

2. **AI agent의 정의 + 패러다임 전환** ⭐
   - 기존 업무 방식과 뭐가 다른가
   - **여기에 오프닝 데모(Ex1)를 실증으로 배치**
     - *"이 발표자료도 AI로 만들었다"* — 이 슬라이드를 만든 대화 과정 자체를 애니메이션으로
     - 형식: 왼쪽에 대화가 오가고, 오른쪽에서 슬라이드 얼개가 한 줄씩 자라나는 그림 (A안)
     - 재생: 자동 재생, 시선을 끄는 수준
     - 역할: 단순 훅이 아니라 **"일하는 방식이 바뀐다"는 개념의 증명**
     - ※ 구체적 표현/디자인은 추후

3. **왜 지금 우리가**
   - 바뀔 때 vs 안 바뀔 때, 변화의 필요성 어필

---

## 본문 1 — 사내/사외 정보 흐름 : "우리 환경부터"

- 사내 서버 LLM (API · OpenCode) vs 사외 Claude Code (도입 예정)
- **정보가 사내에 머무는가, 사외로 나가는가**를 시각화
- 보안 엄격한 환경이라 팀원들이 가장 먼저 궁금해할 지점 → 본문의 문을 여는 자리
- 이후 나올 부품·harness 얘기의 전제 배경이 됨

---

## 본문 2 — plugin / skill / mcp : "부품을 이해한다"

1. **개념·동작원리**
   - plugin · skill · mcp가 각각 뭔지, 어떻게 작동하는지
   - 도식·비유 중심으로 쉽게 (레고 블록 비유 등)

2. **간단한 동작 예시 (Ex2)**
   - agent와 skill이 실제로 이렇게 움직인다, 눈으로 바로 확인
   - 개념을 들은 직후 체감시키는 자리

3. **유용한 것들 소개 + 설치법 + 주의사항**
   - 대표 skill · mcp · plugin — **1장에 압축**. 3칸(Context확장 / Web / Token절약) × 각 대표 1개.
     - 카테고리 지도(어떤 결의 도구들이 있나)만 본문에 전달, 도구명 나열은 지양
     - 전체 후보 목록은 **부록 B**로 (발표 시간 비용 0, 세미나 후 자가학습 경로)
       - Context 확장: ralph loop, claude-mem, superpowers
       - Web 개발: firecrawl, frontend design, playwright
       - Token 절약: caveman, ponytail, yagni
   - 설치 방법
   - 보안 주의사항 — 별도 섹션 없이 여기서 **간단히만** 언급

---

## 본문 3 — harness : "틀로 감싸서 제대로 쓴다"

1. **harness란 + 작동방식**
   - 앞의 부품(plugin·skill)들이 이렇게 조합·오케스트레이션된다
   - 앞 내용에 계속 기대서 설명 (블록 → 조립 설명서+작업대)

2. **장점과 단점**
   - 뭘 얻고 뭘 치르는지 균형 있게

3. **harness 유무 성능 차이 (Ex4)** — 클라이맥스
   - 있을 때 vs 없을 때

---

## 마무리 — 보류

## 확정 / 미결 / 보류 정리

**확정된 것**
- 핵심 서사: agent 소개 → 장점 → 활용법, 하나의 큰 맥락
- 본문 세 파트: 사내/사외 정보 흐름 → plugin/skill/mcp → harness
- **사내/사외 정보 흐름을 본문의 첫 섹션(본문 1)으로 배치**
- **본문 2 내부 순서: 개념 → 예시 → 소개·설치**
- 오프닝 데모(Ex1)를 도입 2번(정의·패러다임)으로 이동, 개념의 실증으로 승격
- Ex1 애니메이션은 A안(대화 재생 + 얼개 성장), 자동 재생
- GitOps 협업(Ex3) 제외
- 설치 주의사항은 본문 2에 흡수, 간단히만
- **본문 2 "유용한 것들"은 1장(카테고리 3칸 × 대표 1개), 전체 목록은 부록 B**

**아직 안 정한 것**
- 마무리 파트 전체 방향

**보류**
- 마무리 파트
- 모든 시각적 표현·디자인·애니메이션 구체화
