# AI Agent 기술 세미나 — 발표 시놉 (v1.8)

## 기본 정보

- **목표**: 청중이 ① LLM을 대략 이해하고 ② AI Agent가 무엇인지 알고 ③ Skill/MCP/Harness의 역할을 구분하고 ④ 세미나 후 **자신이 바로 검증 가능한 작은 업무 하나**에 Agent를 적용해 보게 하기
- **청자**: 반도체 설계/검증/소프트웨어 엔지니어. AI Agent 처음이거나 ChatGPT 정도만 경험
- **발표 시간**: **90분**
- **핵심 메시지(고정)**: **"AI Agent는 사람의 판단을 대체하는 도구가 아니라, 판단 전에 필요한 반복 작업을 줄이는 도구다."**
- **관통 시나리오(v1.8)**: **Regression Triage Agent** — 하나의 작업(regression 실패 분류)을 데모 1(prompt만) → 데모 2(Skill) → 데모 3(MCP) → 데모 4(Harness) → 사람 승인으로 단계적으로 키운다. 회의 메모 등 다른 예시는 보조·부록.
- **4요소(v1.8, 3장에서 동등 4행으로 시각화 — 트리 폐기)**: Agent(생각하고 실행 — LLM+도구+루프) / Skill(일하는 방법·규칙) / MCP(일할 데이터·도구 연결) / Harness(AI가 실수해도 안전하게 — 검증·권한·로그·승인). 질문 4 — 어떻게 움직이나 · 무엇으로 일하나 · 무엇과 연결되나 · 어떻게 통제하나.
- **Agent 도입 4원칙(v1.8, 12장에서 1회 정의 후 배지로 반복)**: ① 작게 시작한다 ② 출력을 구조화한다 ③ 근거를 남긴다 ④ 사람이 승인한다
- **(v1.9, 2026-07-10) 디자인 전면 점검**: Master Visual(Harness=외곽 경계) 7개 장 재사용 · 움직이는 배경은 표지·간지·클로징만 · CORE 카드 축소(과정·리본·계단·결정 흐름·스펙트럼형 전환) · 발표 모드 `?mode=presentation`(CORE+DEMO 37장, READ/APPENDIX·reader-only 제외) · 26장 MCP 통제 READ→CORE(위험 3종) · 부록 49 분할+52·53 병합(총 54장 유지)
- **(v1.8.1, 2026-07-10) 전 장 카피 에디팅**: 하단 note 1~2문장(예고 삭제) · 지시어 구체화 · 과장 완화(표지 "작게 시작", "완수"→"진행") · 용어 한국어화 · 실패 ③ 추가 · L0~L5 재정의(읽기 전용 조회/승인 후 반영/제한된 자동 실행) · 설치 checksum 절차 일관화 · 도구 후보 표 전환
- **핵심 서사(척추)**: *"알고, 써보자"* — 3장에서 결론, 4장에서 즉시 데모(발표 5분 내), 5장이 정의로 접는다("위 과정을 하나로 묶으면 AI Agent의 정의") — 훅 질문("무엇이 더 필요한가")은 발표자 멘트로
- **톤**: "알아서 다 해준다" 금지 — "정해진 권한과 한도 안에서". v1.8 추가 보수화: LLM "단 하나"→기본 동작+결합 · 환각="구조적으로 나타날 수 있는 실패 모드" · 컨텍스트 초과="구성에 따라 제거·요약·실패" · Skill 로드="지원하는 클라이언트가, 방식은 클라이언트마다" · L0~L5="이 세미나의 실무용 예시(외부 표준 아님)" · harness 판단="5질문 체크"(이분법 폐기) · 외부 데이터="통제 불가"→"보관·접근·사용 조건이 다를 수 있다"
- **표기(v1.8 변경)**: 본문 텍스트는 **Agent · Skill · Harness · MCP**(대문자)로 통일 — 단 경로(`.opencode/skills/`)·파일명(`SKILL.md`)·명령어(`opencode`)는 원형 유지. (구 v0.9 "skill 소문자" 규칙 대체)
- **업무환경**: 사내 직접 서빙(OpenCode · Roo · DS assistant API — 데이터 사내 유지) / 사외 Claude(도입 예정 — 외부 전송)
- **자료형태**: reveal.js 단일 파일 HTML(vibe 테마) / autoshow — **54장** (본편 40 + 부록 14). 레이어 배지: DEMO/READ/APPENDIX(CORE 무표기)
- **발표 대본**: [speaker-notes.md](speaker-notes.md) — 장별 핵심 문장·오해 바로잡기 6·90분 배분

### 90분 배분 (v1.8)

결론·4요소(1–3, ~5분) → 데모 1+훅(4–5, ~6분) → 정의·기초(6–10, ~11분) → 맡길 일·4원칙(11–12, ~5분) → 경계(13–16, ~8분) → Skill(17–22, ~17분) → MCP(23–26, ~10분) → Harness(27–35, ~16분) → 마무리(36–40, ~12분)
