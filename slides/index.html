도입 · 정의

AI agent란?

목표를 주면, 스스로 도구를 골라 여러 단계를 밟아 일을 끝내는 AI. 한 번의 질문·답변이 아니라, 확인하고 고치며 끝까지 진행합니다.

기존 방식

내가 한 단계씩

파일 열고 · 찾고 · 고치고 — 매 단계를 사람이 지시.

agent 방식

의도를 말하면

"이걸 이렇게 해줘" → 알아서 도구를 쓰고 결과를 확인·수정.

핵심 3가지

도구·다단계·검증

도구를 쓴다 / 여러 단계를 스스로 / 결과를 확인하고 고친다.

본문 1 · 데이터 경계

사내에 머문다

사내 LLM(API·OpenCode). 설계·검증 데이터가 회사 경계를 벗어나지 않음.

본문 2 · 부품을 이해한다

부품 셋 — skill · mcp · plugin

skill

모델이 호출하는 능력

재사용 절차·전문지식을 SKILL.md로 정의 — 요청과 맞으면 모델이 필요할 때 스스로 로드.

mcp

외부 연결 표준 프로토콜

AI를 외부 도구·데이터에 잇는 개방형 규격(client–server). 서버가 tools·resources·prompts를 노출 — "AI용 USB-C".

plugin

확장 배포 패키지

skill·MCP·hook을 묶어 버전관리·설치하는 자기완결 꾸러미. marketplace에서 설치.

본문 2 · 동작 원리

어떻게 동작하나

skill · 로드

필요할 때만 뜬다

① 평소엔 이름·설명만 상주 → ② 요청과 맞을 때 본문·스크립트 로드 (progressive disclosure) — 컨텍스트 절약.

mcp · 연결

붙이고 골라 호출

① 서버 연결·tool 목록 교환 → ② 모델이 필요한 tool을 골라 호출·결과 수신. 전송: 로컬(stdio)·원격(HTTP).

plugin · 배포

묶어서 설치

① 마켓플레이스 등록 → ② 설치 시 구성요소가 plugin명:이름 네임스페이스로 자동 등록.

관계: plugin은 담아 배포하는 꾸러미, skill·mcp는 그 안에 담기는 능력·연결(단독으로도 씀). — 다음 장에서 skill이 실제로 움직이는 걸 봅니다.

본문 3 · 틀로 감싸 제대로

harness — 모델을 감싸는 실행 틀

skill·mcp를 정해진 순서·규칙으로 묶고, 출력을 검증하고, 실패하면 다시 돌려 항상 같은 품질로 수렴시키는 실행 틀. — DUT를 감싸 정해진 자극·검사를 가하는 verification harness와 같은 결, 대상이 모델일 뿐입니다.

skill·mcp+순서·규칙+검증·재시도=재현·검증되는 실행

본문 3 · 장점과 단점

얻는 것

일관된 품질 · 재현성
