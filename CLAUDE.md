- 인터랙션: 대부분 장은 `.autoshow`(진입 시 자동 스태거), **fragment는 3곳만**(37장 harness 유무 대조, 38장 회고 펀치라인, 41장 B 지평선+지지대). **데모는 전부 정적**(오토플레이 없음 — 커스텀 JS는 hero 배경·섹션 푸터 sync만). 예외: **기초 4~7장(예측·컨텍스트·토큰·환각)은 슬라이드 진입 시 1회 재생 후 정지**(`section.present` 게이트 + fill forwards, `llm*`/`ctx*`/`tok*`/`hal*` keyframes) + 패널 아래 "↻ 다시 재생" 버튼(`simReplay` JS — sim-reset 토글). **10장(agent 등식·루프)만 무한 순환**(`agPulse`).

- harness = **verification harness(UVM)** 비유 + 공식 근거("agentic harness"). 33장 병렬 다이어그램(sequence→DUT→assertion ↔ 입력·절차→모델→스키마 검증). 스키마=assertion(형식 게이트), 내용 정합은 사람이(CTA 검수 루프와 연결).
- **17장 격리 다이어그램**: 사용자 노드 두 갈래 분기(사내 초록/사외 경고색, 빗금 경계 벽). **보안 원칙 1회 온전 진술 + 콜백만.** 19장 판단 사례엔 **"정확한 기준은 팀 보안 규정이 우선" 문구 필수.**
- **14장 "일의 재배치"**: 기술(記述) 톤 — 척추 A 유지. GitOps 소재 제외. 마무리 골격(회고 → FAQ → CTA → Q&A).

- **데모 예시는 "구성된 illustrative 사례"** (`docs/seminar-page-list.md` 부록 A). 참인 메커니즘을 대표 사례로 보여주는 것 — 그대로 제시해도 정당함. 실패 3종(필드 누락/키 드리프트/preamble)은 벤더 공식 문서가 인정하는 실제 현상. 데모 1·데모 3에도 "구성된 예시 장면" 라벨 명시(데모 3의 이슈 ID는 가상).

- 재현성 주장은 "스키마 일치/같은 구조"까지만 — "매번 동일한 JSON" 같은 값 결정론 표현 금지 (재발 주의).
- 사내 도구(18장)·판단 사례(19장) 세부는 사용자 확인 전까지 일반적 서술 유지 — 사내 서비스 실제 사양을 지어내지 말 것. 데모 1 대화의 디테일(로그 3파일·12건 등)도 illustrative — 팀 용어와 어긋나면 교체. 용어는 팀 실사용 기준(예: "회귀"가 아니라 영문 "regression").
